# Промпт: анализ отзывов (3 категории)

Роль и правила достоверности — см. системный промпт (00_system_base).

Задача: классифицировать переданные отзывы (о своём объекте или о конкурентах) и обобщить.

Категории (ровно 3):
- Положительный — доволен, хвалит, без конкретных претензий.
- Отрицательный — недовольство, жалоба, низкая оценка.
- Конструктивный — есть конкретное предложение/замечание, что улучшить (самая ценная категория).

Для каждого отзыва верни: категория + суть одной строкой + тег темы (чистота, заселение, цена, шум, персонал, локация, оснащение и т.п.).

Что сделать в сводке:
1. Всегда явно указывай период сравнения: какой отчётный период с каким базовым сравнивается (например: «10.07.2026–16.07.2026 против 03.07.2026–09.07.2026» или «неделя 10–16.07 против 03–09.07»). Если базового периода нет — напиши «сравнение недоступно (нет данных за прошлый период)» и не вычисляй динамику вслепую.
2. Сравнение с конкурентами веди только за один и тот же период (не смешивай разные окна).
3. Дай распределение по 3 категориям (числа и %), с динамикой к базовому периоду.
4. Топ-3 темы жалоб и топ-3 темы похвал.
5. 1–3 самых важных конструктивных замечания дословно (обезличенно).

Формат ответа (строго):
- Период: <отчётный период против базового периода>
- Распределение: <положительные / отрицательные / конструктивные — числа, %, Δ к базовому>
- Темы: <топ жалоб; топ похвал>
- Конструктив: <1–3 обезличенные цитаты или «нет»>
- Конкуренты: <сравнение репутации за тот же период или «нет данных»>

Правила: не выдумывай отзывы и темы, работай только с переданным текстом. Личные данные (имена, телефоны) не выводи. Если тональность неоднозначна — выбирай «Конструктивный» при наличии предложения, иначе по преобладающему тону. Даты — ДД.ММ.ГГГГ (см. README промптов).
