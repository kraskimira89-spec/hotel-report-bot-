# Промпт: рекомендации (действия для роста дохода)

Роль и правила достоверности — см. системный промпт (00_system_base).

Задача: на основе аналитики цифр и/или анализа отзывов предложить конкретные действия.

Правила:
1. Рекомендации только на основе переданных данных и выводов. Никаких общих советов «из воздуха».
2. Каждая рекомендация должна ссылаться на факт-основание (какая цифра/отзыв её вызвал).
3. Приоритезируй по влиянию на доход и простоте внедрения.
4. Формулируй как действие: что сделать, для чего, ожидаемый эффект (если данные позволяют оценить).
5. Отделяй уверенные рекомендации от гипотез (пометка «проверить»).
6. Не повторяй одну и ту же мысль. Критерий успеха, откат и шаги пилота — каждый пункт один раз,
   без копирования между блоками «что происходит / шаги / проверка / если нет результата».

Формат ответа:
- Рекомендация 1: <действие> — основание: <факт/цифра> — эффект: <ожидание или «нужно проверить»>
- Рекомендация 2: …
- (максимум 3–5 пунктов, самые важные сверху; без дублей формулировок)

Если данных недостаточно для обоснованной рекомендации — так и напиши, не выдумывай.

## Внешние тренды и адаптация для Томска

Если во входных данных передан список `external_trends`, анализируй его на трёх уровнях:
1. `world` — мировые практики и технологии.
2. `moscow` / `spb` — практики наиболее развитых гостиничных рынков России.
3. `tomsk` / `siberia` — локальные и региональные подтверждения.

Мировой и столичный тренд не является готовой рекомендацией для 1apart.
Он является гипотезой о возможном будущем спросе или изменении рынка Томска.

Для каждого релевантного внешнего тренда:
- укажи название, источник, ссылку и дату публикации;
- отдели подтверждённый факт от предположения;
- оцени применимость к апартаментам Томска: высокая / средняя / низкая;
- объясни, почему практика может прийти в Томск с задержкой;
- не выдумывай срок внедрения. Если нет локального подтверждения, пиши:
  «срок распространения в Томске требует проверки»;
- предложи малозатратный пилот вместо масштабного внедрения;
- укажи метрику успеха и срок проверки пилота.

Не предлагай дорогую автоматизацию, изменение тарифов или закупку технологии,
если нет местных данных, расчёта окупаемости либо результатов пилота.

Не добавляй тренды, которых нет в `external_trends`. Если список пуст — не предлагай
мировые или московские практики «из знаний модели».

Формат пилотной рекомендации по тренду (каждый пункт один раз, без повторов):
```text
Тренд: …
Уровень: … → потенциальная адаптация для Томска.
Факт: практика подтверждена источником от ….
Гипотеза: …
Применимость к 1apart: высокая / средняя / низкая.
Пилот: …
Метрики: …
Условие масштабирования: …
```
Не копируй «Условие масштабирования» и текст отката («остановить пилот…») в другие блоки ответа.
