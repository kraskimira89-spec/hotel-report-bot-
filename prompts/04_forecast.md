# Промпт: прогноз загрузки, выручки и рекомендации по ценам

Роль и правила достоверности — см. системный промпт (00_system_base).

Задача: объяснить владельцу прогноз операционных и финансовых показателей апарт-отеля и что сделать с ценами — на основе переданных данных модели (не выдумывать цифры).

## Входные данные (что может быть в JSON)

- Горизонт: **7 / 14 / 30 / 180** дней.
- Сценарий: **базовый**, **консервативный**, **оптимистичный**.
- Метрики по дням/неделям: загрузка %, ADR, RevPAR, выручка, проданные unit-nights, факт (если уже известен), нижняя/верхняя граница.
- Факторы модели (`factors_json`): сезонность, день недели, pickup, уже забронировано, корректировка по рынку, объём истории, заметки.
- Рекомендации по ценам: тип (повысить / оставить / снизить / ограничить скидки / ручная проверка), текущая и рекомендуемая цена, причина, уверенность.
- Качество прогноза: MAE загрузки, MAPE выручки, предупреждения.
- Медиана цен конкурентов (если есть).

## Правила интерпретации

1. **Приоритет фактов:** уже забронированные даты важнее pickup и сезонности.
2. **Горизонт 7–14 дней** — можно говорить конкретнее (оперативные цены, ближайший спрос).
3. **Горизонт 30 дней** — план выручки и загрузки; допускай небольшую неопределённость.
4. **Горизонт 180 дней** — **не давай одну точную цифру** как финальный прогноз. Описывай **диапазон** и **сценарии** (базовый / консервативный / оптимистичный). Явно укажи, что точность ограничена историей и будущими событиями.
5. **Уверенность:** высокий / средний / низкий — объясни почему (объём истории, горизонт, качество модели).
6. **Рекомендации по ценам** — только рекомендации, без автоматического изменения тарифов в TravelLine. Если данных мало — «ручная проверка», не «снизить/повысить».
7. Не маскируй низкую достоверность: честно пиши «данных недостаточно для уверенного прогноза».

## Что сделать в ответе

1. Укажи **период прогноза** (даты ДД.ММ.ГГГГ), **горизонт** и **сценарий**. Если сравнение с прошлым фактом недоступно — «сравнение недоступно».
2. **Главный вывод** одной фразой: что ждать по загрузке и выручке.
3. **Ключевые цифры** из данных: средняя/итоговая загрузка, ADR, RevPAR, выручка (с границами, если есть).
4. **Почему такой прогноз** — 2–4 фактора из `factors_json` простым языком.
5. **Цены:** 1–3 самые важные рекомендации (дата, категория, действие, причина).
6. **Риски и ограничения** — если уверенность низкая или MAE/MAPE выше нормы.

Формат ответа (строго):
- Период: <даты прогноза, горизонт N дн., сценарий>
- Вывод: <одна фраза>
- Прогноз: <загрузка, ADR, RevPAR, выручка — только из данных; для 180 дн. — диапазон/сценарии>
- Факторы: <2–4 пункта>
- Цены: <топ рекомендаций или «нет активных»>
- Уверенность: <высокий|средний|низкий> — <почему>
- Внимание: <риски, качество модели или «нет»>

Никаких выдуманных чисел. Даты — ДД.ММ.ГГГГ.
