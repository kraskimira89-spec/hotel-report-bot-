# Как разработчику видеть задачи GitHub в Cursor

Три рабочих способа — от простого к продвинутому. Для нашего проекта достаточно Способа 1.

---

## Способ 1 — файлы-промпты в проекте (рекомендуется, ничего настраивать не надо)

Все задания уже лежат в репозитории в папке `docs/`, поэтому в Cursor они видны как обычные файлы проекта.

Что делает разработчик:
1. Открывает проект `hotel-report-bot` в Cursor.
2. В дереве файлов слева открывает нужный промпт из `docs/`, например:
   - `docs/Cursor_Промпт_ИИ_лента.md` (задача #2)
   - `docs/Cursor_Промпт_Редизайн.md` (задача #3)
   - `docs/Cursor_Промпт_Разделы_Конкуренты_Тренды.md` (задача #4)
   - `docs/Этап_6-7_Автосбор_виджетов.md` (задача #5)
3. Копирует текст промпта (в файлах он после строки `====`).
4. Открывает чат **Agent / Composer** (Cmd/Ctrl+I) и вставляет промпт → запускает.

GitHub Issues (#2–#5) при этом служат «оглавлением» бэклога: что делать и в каком порядке. Сами инструкции — в файлах.

Связь issue ↔ файл: в каждом issue есть ссылка на соответствующий файл-промпт.  
В Cursor правило: `.cursor/rules/github-tasks.mdc` (агент знает эту таблицу).

---

## Способ 2 — расширение «GitHub Pull Requests and Issues» (видеть список задач в редакторе)

Cursor основан на VS Code, поэтому подходит официальное расширение GitHub.
1. Слева панель **Extensions** (значок кубиков) → найти **GitHub Pull Requests and Issues** → **Install**.
2. Внизу/слева нажать **Sign in to GitHub** → авторизоваться.
3. Появится панель **Issues** со списком задач репозитория — по клику открывается описание.
4. Можно правой кнопкой по issue → «Start working on issue» (создаёт ветку).

Плюс: список задач прямо в редакторе. Минус: нужна авторизация в GitHub.

---

## Способ 3 — GitHub MCP (Cursor-агент сам читает Issues)

Позволяет писать в чате: «Прочитай issue #2 из репозитория и реализуй по описанию».

1. Создать GitHub Personal Access Token: GitHub → Settings → Developer settings → Personal access tokens → Fine-grained token, доступ к репозиторию `hotel-report-bot-` (права: Contents, Issues — чтение).
2. Скопировать шаблон:
   ```bash
   cp .cursor/mcp.json.example .cursor/mcp.json
   ```
   и заменить `ВАШ_ТОКЕН` на токен.
3. Cursor → Settings → MCP → убедиться, что сервер `github` включён (зелёный).
4. В чате Agent: «Возьми issue #2 из kraskimira89-spec/hotel-report-bot- и выполни».

⚠️ Токен — секрет. Файл `.cursor/mcp.json` в `.gitignore`, не коммитить. В репозитории только `.cursor/mcp.json.example`.

Плюс: агент сам подтягивает задачи. Минус: настройка токена, чуть сложнее.

---

## Что выбрать
- Быстро и без настройки → **Способ 1** (файлы уже в проекте).
- Хочется видеть список задач в редакторе → **Способ 2**.
- Хочется, чтобы агент сам читал issues → **Способ 3**.
