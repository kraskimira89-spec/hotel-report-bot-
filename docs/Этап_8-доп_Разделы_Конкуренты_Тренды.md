# ТЗ (дополнение к этапу 8) — новые разделы админки «Конкуренты» и «Тренды»

Документ для разработчика к ТЗ v2.2. Добавляет два раздела в веб-интерфейс (FastAPI-админка). Составлен на основе текущего состояния репозитория (июль 2026).

---

## Контекст: что уже есть в админке

Навигация (`src/web/templates/base.html`): Дашборд · Цены · Метрики · Каналы · Отчёты · Логи · Настройки. Маршруты в `src/web/app.py`. Данные — из SQLite через `src/web/queries.py`. Конкуренты уже описаны в `config.competitors` (поля name, type, url, parser, selectors), сбор — в `src/data_sources/market_trends.py` (`CompetitorPriceInfo`).

Задача: добавить **два новых пункта меню и две страницы** — «Конкуренты» и «Тренды», не ломая существующие.

---

## РАЗДЕЛ 1. «Конкуренты» (`/competitors`)

### 1.1 Цель
Единая страница со всей информацией о конкурентах: карточки объектов, их публичные цены (история), сравнение с ценами 1apart.ru, скриншоты виджетов (доказательство), статус сбора.

### 1.2 Маршрут и навигация
- Добавить пункт меню «Конкуренты» в `base.html` (между «Каналы» и «Отчёты»).
- Маршрут `@app.get("/competitors")` в `app.py`, шаблон `templates/competitors.html`, `page="competitors"`, защита `_require_auth`.

### 1.3 Содержание страницы

**Блок А — Обзорная таблица (все 9 конкурентов):**

| Колонка | Источник данных |
|---|---|
| Название | config.competitors.name |
| Тип | direct / indirect (значок/цвет) |
| Движок сбора | parser (static / tl_widget / wubook_widget) |
| Текущая цена «от» | последний CompetitorPriceInfo.price_from |
| Δ к нашей цене | сравнение с ценой 1apart сопоставимой категории |
| Обновлено | дата последнего успешного снимка |
| Статус | 🟢 собрано / 🟡 из кэша / 🔴 недоступно (available) |
| Ссылка | url (внешняя, target=_blank) |

**Блок Б — Карточка конкурента (по клику / раскрытие):**
- Полное название, тип, адрес сайта.
- История цен: мини-график/таблица по датам (из БД competitor_prices за 90 дней).
- Скриншот виджета за последнюю дату (из `data/screenshots/competitors/…`), с датой снимка.
- Способ получения цены: dom / vision (пометка достоверности).
- Категорийное сопоставление: какие типы номеров конкурента сравниваются с нашими 6 категориями.

**Блок В — Сводка «Мы vs рынок»:**
- Средняя цена прямых конкурентов, наша цена, позиция (выше/ниже/в рынке) в %.
- Отдельно прямые (Петровские, Bon Apart, Центральный, Гоголь) и косвенные (Xander, Кухтерин, Скандинавия, Парад Парк, Элегант).

### 1.4 Данные и БД
- Читать из таблицы `competitor_prices` (если её нет — создать: id, competitor_name, date, price_from, currency, source(dom|vision), screenshot_path, available, created_at).
- Скриншоты отдавать статикой (FastAPI `StaticFiles`, маршрут `/static/screenshots/…`) — только для авторизованных.
- Запрос-хелпер в `queries.py`: `get_competitor_latest()`, `get_competitor_history(name, days=90)`.

### 1.5 Требования к коду
- Никаких секретов/персональных данных на странице.
- Если цена недоступна (`available=False`) — показывать «нет публичной цены», не пусто.
- Сопоставление категорий — через справочник в config (competitor ↔ наши slug), не хардкодом в шаблоне.

### 1.6 Тесты
- `get_competitor_latest`/`get_competitor_history` на фикстурной БД.
- Рендер `/competitors` под авторизацией = 200; без авторизации = редирект на /login.

## РАЗДЕЛ 2. «Тренды» (`/trends`)

### 2.1 Цель
Раздел, который автоматически отслеживает и показывает лучшие идеи и тренды в сфере апарт-отелей — по Москве/России и за рубежом. Это «радар идей»: что внедряют лидеры рынка, какие технологии и практики повышают доход и загрузку.

### 2.2 Маршрут и навигация
- Пункт меню «Тренды» в `base.html` (после «Конкуренты»).
- Маршрут `@app.get("/trends")`, шаблон `templates/trends.html`, `page="trends"`, защита `_require_auth`.

### 2.3 Содержание страницы

**Блок А — Лента трендов (карточки):**
Каждая карточка: заголовок · краткое описание (1–2 предложения) · категория · регион (🇷🇺 Россия / 🌍 Мир) · источник (ссылка) · дата · практический вывод «что это значит для нас».

**Блок Б — Фильтры:** по региону (Россия / зарубеж / все), по категории, по дате (последние 7/30/90 дней).

**Блок В — «Идея недели»:** одна выделенная карточка — самая релевантная идея для внедрения.

### 2.4 Категории трендов (фиксированный справочник)
1. **Технологии и ИИ** — чат-боты, ИИ-ассистенты, агентные системы (сами отвечают гостю, двигают цену в заданных лимитах), разговорная аналитика.
2. **Динамическое ценообразование / RMS** — revenue management, объяснимые рекомендации цены.
3. **Прямые бронирования** — снижение зависимости от агрегаторов, лояльность, прямой канал.
4. **Бесконтактный сервис** — онлайн-заезд, смарт-замки, самообслуживание.
5. **Длительное проживание / корпоративные гости** — рост спроса на long-stay.
6. **Гость-опыт и допуслуги** — персонализация, апселл.
7. **Рынок и регулирование** — спрос, цены, законы (РФ: классификация, реестр).

### 2.5 Источник данных трендов (автоотслеживание)
Расширить `src/data_sources/market_trends.py`:
- Функция `fetch_market_trends(period, region)` → список `TrendItem`.
- **Способ сбора (гибрид, автоматический, 1 раз/неделю):**
  - **RSS/новостные ленты отраслевых порталов** (гостеприимство, апарт-отели) — httpx + парсинг RSS/HTML, те же анти-блок правила.
  - **Периодический ИИ-дайджест** — на основе собранных заголовков/аннотаций формировать 5–8 карточек с категорией, регионом и практическим выводом (LLM-суммаризация).
- Источники задать в `config.market_news.sources` (список RSS/URL) + `config.market_news.regions`.
- Хранить в БД (`trends`: id, title, summary, category, region, source_url, published_at, takeaway, is_idea_of_week, created_at) — ретеншен 180 дней.

### 2.6 Структура данных
```python
class TrendItem(BaseModel):
    title: str
    summary: str
    category: str        # из справочника 2.4
    region: str          # ru | world
    source_url: str
    published_at: date | None
    takeaway: str        # "что это значит для нас"
    is_idea_of_week: bool = False
```

### 2.7 Стартовое наполнение (проверенные тренды 2026 — заложить как сиды)
Актуальные тренды из авторитетных источников (для первичного наполнения БД):

- **Агентный ИИ в отелях.** ИИ перешёл от «подсказок» к самостоятельным действиям: отвечает гостю, обрабатывает заявку, двигает тариф в заданных лимитах; человек — только исключения. Категория: Технологии и ИИ. Регион: world. Вывод: снижает трудозатраты и «ловит» выручку в нерабочее время. Источник: Hotel Tech Report Q2 2026 (https://hoteltechreport.com/news/q2-2026-hotel-ai-trends-report).
- **Разговорная аналитика.** Вместо дашборда — вопрос простым языком и мгновенный ответ; данные попадают в решение вовремя. Категория: Технологии и ИИ. Регион: world.
- **Объяснимое ценообразование.** RMS показывает, почему рекомендует цену (сигналы спроса) — растёт доверие и применение. Категория: Динамическое ценообразование. Регион: world.
- **Рост рынка serviced apartments.** Мировой рынок апарт-отелей: ~$132 млрд (2025) → прогноз ~$434 млрд к 2033, CAGR 16,9%; прямые брони ~44,9% рынка, long-stay доминирует. Категория: Рынок. Регион: world. Источник: Grand View Research (https://www.grandviewresearch.com/industry-analysis/serviced-apartment-market-report).
- **Москва: дефицит предложения, рост цен.** В 1П2026 сделки с апартаментами в Москве снизились из-за запрета новых проектов, предложение упало почти вдвое, средняя цена за м² выросла на 58,7%. Категория: Рынок и регулирование. Регион: ru. Источник: РНРФ (https://rnrf.ru/news/24413-spros-i-ceny-na-apartamenty-v-moskve-izmenilis-v-2026-godu/).
- **Приоритет прямых бронирований.** Снижение зависимости от агрегаторов через лояльность и прямой канал — устойчивый тренд. Категория: Прямые бронирования. Регион: world/ru.
- **Бесконтактный заезд и смарт-замки.** Онлайн-регистрация и самостоятельный заезд как стандарт апарт-формата. Категория: Бесконтактный сервис. Регион: world/ru.

### 2.8 Требования к коду
- Тренды собираются отдельной еженедельной задачей планировщика (не в ежедневном цикле).
- Все внешние ссылки — с указанием источника и даты (для доверия).
- «Идея недели» выбирается по релевантности (например, по категории, приоритетной для заказчика — из config).
- Graceful-фолбэк: если источники недоступны — показать последние сохранённые тренды + сиды из 2.7.

### 2.9 Тесты
- `fetch_market_trends` с моками RSS → возвращает список `TrendItem`.
- Рендер `/trends` = 200 под авторизацией; фильтры по региону/категории работают на фикстурной БД.
- Сиды из 2.7 загружаются при пустой БД.
