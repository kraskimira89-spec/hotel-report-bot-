# Этапы 6/7 — аудит сайтов конкурентов и селекторы для сбора цен

Документ для разработчика к ТЗ v2.2. Аудит проведён на реальной верстке (июль 2026). Цель — пройти сбор цен конкурентов без проб и ошибок.

Правила (ТЗ v2.2): цены конкурентов идут ТОЛЬКО в еженедельный email-отчёт (справочно). Соблюдаются те же анти-блок правила, что и для 1apart.ru (паузы 2–3 сек, случайные задержки, реалистичный User-Agent, уважение robots.txt, backoff 403/429/503). Загрузку конкурентов из открытых источников надёжно получить нельзя — только цены, где они публично доступны.

## Селекторы для сайтов со статическими ценами

### 1. Петровские (Tilda) — apartment.tomsk.ru
- Название: `.js-product-name` (или `.t-name`)
- Цена: `.t776__price-value` (он же `.js-product-price`)
- Формат значения: строка с несколькими тарифами, напр. «3-е суток - 26 000 руб| 2-е суток - 18 000 руб| 1 сутки - 12 000 руб» — нужен разбор по разделителю `|` и извлечение «1 сутки».
```python
for card in soup.select("[class*='t776__product']"):
    name = card.select_one(".js-product-name")
    price = card.select_one(".t776__price-value, .js-product-price")
    # из текста цены выделить '1 сутки - N руб'
```

### 4. Гоголь — gogolhotel.ru
- Цена в тексте: паттерн «Цена от N руб» внутри `<p>` (блок `#order-online` / карточки номеров).
- Собирать regex: `Цена от\s*([\d\s]+)\s*руб`.
```python
import re
m = re.search(r"Цена от\s*([\d\s]+)\s*руб", html)
price = int(m.group(1).replace(" ", "")) if m else None
```

### 6. Кухтерин — kuhterinhotel.ru

## Сайты с ценами только в виджете (6 из 9)

Bon Apart, Центральный, Xander, Скандинавия, Парад Парк, Элегант — цены отдаются JavaScript-виджетами (в основном TravelLine `tlintegration`, а также Bnovo/iframe). В статическом HTML цен нет.

Рекомендация (по аналогии с решением для 1apart.ru):
- **Не парсить виджеты через браузер** — это хрупко и рискованно (риск блокировки).
- Для этих объектов на этапе 6/7 выбрать один из вариантов:
  1. **Оставить без автоцены** — в email показывать «цена уточняется вручную / недоступна в открытом виде» (`available: false`). Минимально надёжно.
  2. **Ручной ввод** через админку (поле «цена конкурента на неделю»).
  3. **Playwright точечно** только для этих 6 сайтов, если нужна автоматика — но это отдельный объём работ и повышенный риск. Обсуждать отдельно.

По умолчанию рекомендуется вариант 1+2: автоматически собирать 3 сайта со статикой (Петровские, Гоголь, Кухтерин), остальные — помечать `available: false` и/или давать ручной ввод.

## Что заложить в код (этап 6/7)

- В `market_trends.fetch_competitor_prices()` перебирать `config.competitors`; для каждого — стратегия по полю `parser` (см. ниже).
- Добавить в конфиг каждому конкуренту поле `parser`: `static` (BeautifulSoup) или `widget` (пропуск/ручной ввод), и опционально `selectors`.
- Переиспользовать анти-блок и backoff из `site_prices`.
- Возвращать структуру: `{name, type, url, price_from, available, source}`; при `widget` → `available: false`.
- Для Элегант — не запрашивать *.html (robots), только корень/разделы.

### Пример дополнения конфига (settings.yaml)
```yaml
competitors:
  - name: "Апартаменты Петровские"
    type: direct
    url: "https://apartment.tomsk.ru/"
    parser: static
    selectors: { name: ".js-product-name", price: ".t776__price-value" }
  - name: "Гоголь"
    type: direct
    url: "https://gogolhotel.ru/"
    parser: static
    selectors: { price_regex: "Цена от\\s*([\\d\\s]+)\\s*руб" }
  - name: "Кухтерин"
    type: indirect
    url: "https://kuhterinhotel.ru/catalog/catalog/"
    parser: static
    selectors: { price_block: ".price" }
  - name: "Bon Apart"
    type: direct
    url: "https://www.bon-apart.ru/"
    parser: widget        # цены в виджете TravelLine — available:false / ручной ввод
  # Центральный, Xander, Скандинавия, Парад Парк, Элегант — parser: widget
```
