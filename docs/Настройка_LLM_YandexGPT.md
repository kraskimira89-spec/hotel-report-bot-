# Подключение LLM через Yandex Cloud (YandexGPT) — Issue #9

Как подключить ИИ для раздела «Аналитика» через YandexGPT. Проверено по документации Yandex AI Studio (2026). Плюсы: российская инфраструктура, оплата в рублях, OpenAI-совместимый API (почти без правок кода).

---

## 0. Что важно понять

- У YandexGPT есть **OpenAI-совместимый эндпоинт** — наш код уже работает в этом формате, нужна лишь мелкая правка (заголовок каталога).
- Нужны 3 вещи: **API-ключ** сервисного аккаунта, **folder_id** (идентификатор каталога), привязанный **billing** (оплата).
- Новым аккаунтам Yandex Cloud даётся грант **4000 ₽ на 60 дней** — на тесты хватит с запасом. Запросы стоят копейки-рубли.

Параметры эндпоинта:
| Параметр | Значение |
|---|---|
| Base URL | `https://ai.api.cloud.yandex.net/v1` |
| Авторизация | заголовок `Authorization: Api-Key <ключ>` |
| Каталог | заголовок `OpenAI-Project: <folder_id>` |
| Модель | `gpt://<folder_id>/yandexgpt/latest` (Pro) или `yandexgpt-lite/latest` (дешевле) |

---

## Шаг 1. Аккаунт и каталог в Yandex Cloud

1. Войдите в [console.yandex.cloud](https://console.yandex.cloud/) под вашим Яндекс ID (тем же, что почта bogdanchik2).
2. Если облака нет — создайте (следуйте подсказкам: организация → облако → каталог).
3. Убедитесь, что есть **каталог** (folder). Обычно называется `default`.
4. **Скопируйте folder_id**: он в адресной строке при выборе каталога, или на странице каталога сверху (строка вида `b1g...`). Сохраните.

---

## Шаг 2. Привязать billing (оплату)

1. Меню → **Yandex Cloud Billing** → создать платёжный аккаунт (нужна карта).
2. Новым аккаунтам активируется грант 4000 ₽. Статус должен быть `ACTIVE` или `TRIAL_ACTIVE`.

Без billing production-квоты не откроются.

---

## Шаг 3. Сервисный аккаунт + роль

1. В консоли → раздел **Identity and Access Management (IAM)** → **Сервисные аккаунты** → **Создать**.
2. Имя: `hotel-report-llm`.
3. Роль: **`ai.languageModels.user`** (доступ к языковым моделям). Добавьте её.
4. Создайте.

---

## Шаг 4. Создать API-ключ

1. Откройте созданный сервисный аккаунт `hotel-report-llm`.
2. Справа вверху **Создать новый ключ** → **Создать API-ключ**.
3. (Область действия, если спросит) — выберите `yc.ai.foundationModels.execute`.
4. Описание: `LLM для аналитики`.
5. Нажмите **Создать**.
6. **Скопируйте секретный ключ** (показывается один раз) и сохраните. Заодно сохраните folder_id из Шага 1.

⚠️ Ключ — секрет, не коммитить в Git.

---

## Шаг 5. Мелкая правка кода (нужна для Яндекса)

Код в `src/analytics/ai_insights.py` (`_call_llm`) сейчас шлёт `Authorization: Bearer` без заголовка каталога. Для YandexGPT добавить заголовок `OpenAI-Project`. Правка минимальна и совместима с любым OpenAI-провайдером.

В функции `_call_llm`, в блоке `headers`, заменить на:
```python
headers = {
    "Authorization": f"Api-Key {api_key}",          # Яндекс: Api-Key вместо Bearer
    "Content-Type": "application/json",
}
folder_id = os.getenv("LLM_FOLDER_ID", "")
if folder_id:
    headers["OpenAI-Project"] = folder_id            # каталог Яндекса
```
(добавить `import os`, если его нет; передавать `headers` в `client.post`).

Примечание: `Api-Key` также работает как `Bearer` у некоторых провайдеров — но для Яндекса корректнее `Api-Key`. Если разработчик хочет универсальность — вынести схему авторизации в конфиг (`LLM_AUTH_SCHEME=api-key|bearer`).

---

## Шаг 6. Настройки в `.env`

```
LLM_API_KEY=<секретный API-ключ из шага 4>
LLM_FOLDER_ID=<folder_id из шага 1>
LLM_BASE_URL=https://ai.api.cloud.yandex.net/v1
LLM_MODEL=gpt://<folder_id>/yandexgpt/latest
```
(имена переменных сверить с тем, как их читает `ai_insights.py`/`config.py`; при необходимости привести к единому виду.)

Для экономии на старте можно взять `yandexgpt-lite/latest` вместо `yandexgpt/latest`.

---

## Шаг 7. Проверка

1. Прописать `.env`, перезапустить сервис.
2. В админке раздел «Аналитика» → кнопка **«Обновить»**.
3. Признак успеха: карточки генерируются осмысленно (не только правила-фолбэк); в логах нет `LLM HTTP 4xx/5xx`.
4. Тестовый curl (для разработчика):
```
curl https://ai.api.cloud.yandex.net/v1/chat/completions \
 -H "Authorization: Api-Key <ключ>" \
 -H "OpenAI-Project: <folder_id>" \
 -H "Content-Type: application/json" \
 -d '{"model":"gpt://<folder_id>/yandexgpt/latest","messages":[{"role":"user","content":"Привет"}]}'
```

## Что нужно от Сергея (итог)
1. folder_id (Шаг 1) + привязанный billing (Шаг 2).
2. API-ключ сервисного аккаунта с ролью `ai.languageModels.user` (Шаги 3–4).
3. Передать разработчику: `LLM_API_KEY`, `LLM_FOLDER_ID` (+ мелкая правка кода из Шага 5).

После этого раздел «Аналитика» начнёт давать настоящие ИИ-выводы, а не только правила-фолбэк.
