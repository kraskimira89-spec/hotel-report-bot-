## 1. Лист «Заселяемость» — структура

Лист = **многолетний архив по годам** (2018 → 2026), идущих сверху вниз. Внутри года — два вида блоков:

### 1.1 Годовые блоки (2018–2025) — МЕСЯЧНАЯ статистика (архив)
```
2025 год                         <- маркер года (строка вида "NNNN год")
Тип квартиры | Кол-во квартир | Январь | .. | Декабрь | Средний %
1-комн.      | 25            | 14,64  | .. | 54,84   | 38,99
1-комн. 23 кв.м. | 6         | ...
1-комн. (улучш.) | ...
1-комн. с диваном | ...
2-комн.      | 11            | ...
общее кол-во | 44            | ...
% из Traveline | ...          <- есть с 2021 года
```

### 1.2 Блоки 2026 года — СУТОЧНАЯ статистика (рабочая, самая важная)
Для 2026 идут помесячные блоки с колонками по ДНЯМ (1..31):
```
ИЮЛЬ                                             <- маркер месяца
Тип квартиры | Кол-во квартир | 1 | 2 | 3 | ... | 31
1-комн. 23 кв.м. | 6 | 5 | 4 | 4 | 4 | ...       <- % заполняемости по дням
1-комн.          | ...
1-комн. (улучш.) | ...
1-комн. (с диванчиком) | ...
2-комн.          | ...
общее кол-во     | 44 | ...
% за день        | ...                            <- итоговый % по дню
% из Traveline   | ...                            <- сверка с TravelLine
```

### 1.3 Важные нюансы
- Значения процентов — **с запятой** («14,64»), надо парсить как float (замена `,`→`.`).
- Прочерк «-» и пустые ячейки = нет данных (None), НЕ ноль.
- Названия типов квартир **менялись по годам** и содержат лишние пробелы («1-комн. » с хвостовым пробелом) — нормализовать (trim, привести регистр).
- Для ежедневной сводки нужны именно **суточные блоки 2026** (раздел 1.2): строка «общее кол-во» и «% за день» на нужную дату.
- Кол-во квартир: 44 всего; по типам в 2026 — см. столбец «Кол-во квартир».

## 2. Лист «Брони статистика» — структура

Лист = **помесячные блоки** (ИЮЛЬ, ИЮНЬ, МАЙ, АПРЕЛЬ...), сверху вниз, обычно текущий и предыдущие месяцы.

```
        | ИЮЛЬ                                              <- маркер месяца (в колонке 1)
        | ИСТОЧНИК БРОНИРОВАНИЯ | | 1 | 2 | .. | 31 | ВСЕГО
        |                       | | ср| чт| .. | пт |        <- строка дней недели
        | Сайт                  | | 3 | 2 | .. |    | 12
        | Уже проживали (по звонку) | | 1 | 4 | .. | | 15
        | Островок              | | .. | 1
        | ... (≈28-31 источник) ...
        | ИТОГО:                | | .. | 38               <- маркер конца блока
```

### 2.1 Реальный список источников (30+, шире, чем в конфиге!)
Сайт, Уже проживали (по звонку), Островок, По звонку (поисковые системы), С улицы, Суточно.ру, Академсервис (АОН.ру), От организации, Система бронирования/Эй Энд Эй, Броневик, 2 ГИС, OneTwoTrip, Отели в Т-банке (Т-Путешествия), От знакомых, Отелло, ОЗОН, Авито, 101 отель, ТВИЛ, Свой Отель, ХБ-ПРО, Тезаурум, Авиа-центр / Отели онлайн, Яндекс Путешествия, Тривио, CBooking, Алеан, Inst, Агрегатор каналов Comfort Booking.

### 2.2 Нюансы
- Источник — в **колонке 2** (индекс 1), значения по дням начинаются с колонки 4 (индекс 3).
- Данные по дням: число броней за этот день по источнику; пусто = 0.
- «ВСЕГО» — последняя колонка блока.
- Маркер конца блока — строка «ИТОГО:».
- Порядок источников между месяцами может немного отличаться — сопоставлять по НАЗВАНИЮ, а не по номеру строки.
- Классификация direct/aggregator — по `channels_map` из config (этап 2), НЕ хардкодить здесь.

### Структуры данных (dataclass / pydantic)
```python
class RoomTypeOccupancy(BaseModel):
    room_type: str        # нормализованное название
    units: int | None     # кол-во квартир
    occupancy_pct: float | None

class OccupancyDay(BaseModel):
    date: date
    by_type: list[RoomTypeOccupancy]
    total_pct: float | None        # строка «% за день» / «общее кол-во»
    travelline_pct: float | None   # строка «% из Traveline»

class BookingSourceDay(BaseModel):
    source: str           # название источника как в таблице
    count: int            # броней за день

class BookingsMonth(BaseModel):
    year: int
    month: int
    by_source: dict[str, int]   # источник -> всего за месяц
    total: int
```

### Алгоритм поиска блоков (ключевое!)
1. Прочитать весь лист как список строк (`worksheet.get_all_values()`).
2. **Заселяемость:** идти сверху, находить маркеры месяцев (ИЮЛЬ/ИЮНЬ/...) в блоке 2026; в найденном блоке взять строку заголовка с днями (1..31), определить индекс колонки нужного дня, затем считать строки типов до «общее кол-во» / «% за день» / «% из Traveline».
3. **Брони:** находить маркер месяца в колонке 1; строку-заголовок с днями; читать источники (колонка 2) до строки «ИТОГО:»; брать значение нужного дня или «ВСЕГО».
4. Никогда не привязываться к абсолютным номерам строк — только к маркерам.

## 4. Требования к коду
- Авторизация: `gspread.service_account(filename=...)`; путь из `.env` (`GOOGLE_SA_JSON_PATH`), сервисному аккаунту дать доступ «Читатель» к таблице.
- gid/названия листов — из `config.sheets` (уже прописаны).
- Парсинг чисел: «14,64» → 14.64; «-»/«» → None; целые для броней.
- Нормализация названий типов и источников: `strip()`, схлопывание пробелов, единый регистр для сопоставления.
- Логирование: если блок/дата не найдены — предупреждение в лог, вернуть пустую структуру (не падать).
- Никакой бизнес-логики (проценты→светофор, каналы) — только чтение и структурирование. Расчёты — этап 2.
