# Пошаговый план настройки SMTP через Google Почту (Gmail)

Инструкция для Сергея (по шагам, куда нажимать). Проверено по актуальным правилам Google (2026). Для не-программиста: делаем медленно, после каждого блока — подтверждение.

Что настраиваем: отправку еженедельного email-отчёта из сервиса через ваш Gmail.

---

## 0. Главное, что нужно знать заранее

⚠️ Обычный пароль от Gmail **НЕ подойдёт** — Google его блокирует для сторонних программ. Нужен специальный **«Пароль приложения»** (App Password) — это 16-значный код только для нашего сервиса.

Чтобы создать «Пароль приложения», у аккаунта должна быть включена **двухэтапная аутентификация (2FA)**. Поэтому порядок такой: сначала 2FA (Шаг 1), потом пароль приложения (Шаг 2).

Рекомендация: заведите отдельный Gmail для отчётов (например, `reports.1apart@gmail.com`), а не личный — так безопаснее. Но можно и существующий.

Официальные настройки Gmail SMTP (2026):
| Параметр | Значение |
|---|---|
| SMTP-сервер | `smtp.gmail.com` |
| Порт (TLS) | `587` |
| Шифрование | TLS |
| Логин | полный адрес Gmail (например, reports.1apart@gmail.com) |
| Пароль | «Пароль приложения» (16 символов), НЕ обычный пароль |

---

## Шаг 1. Включить двухэтапную аутентификацию (2FA)

1. Откройте [myaccount.google.com](https://myaccount.google.com/) (войдите в нужный Gmail).
2. Слева выберите раздел **«Безопасность» / «Security»**.
3. Найдите блок **«Как выполняется вход в аккаунт Google»**.
4. Нажмите **«Двухэтапная аутентификация» / «2-Step Verification»**.
5. Нажмите **«Начать» / «Get started»** и пройдите настройку:
   - подтвердите номер телефона (придёт SMS с кодом),
   - введите код.
6. В конце появится статус **«Двухэтапная аутентификация включена»**.

---

Как включите 2FA — напишите «готово». Дальше Шаг 2 (создание пароля приложения).

---

## Шаг 2. Создать «Пароль приложения»

1. Оставаясь в разделе **«Безопасность»**, в строке поиска вверху страницы аккаунта введите:
   ```
   Пароли приложений
   ```
   (или по-английски **App passwords**) и откройте этот пункт.
   - Прямая ссылка: [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
   - Если пункт не появляется — значит 2FA ещё не включена (вернитесь к Шагу 1).
2. Google попросит подтвердить пароль от аккаунта — введите его.
3. На странице «Пароли приложений»:
   - В поле **«Название приложения» / «App name»** введите:
     ```
     hotel-report-bot
     ```
   - Нажмите **«Создать» / «Create»**.
4. Появится окно с **16-значным паролем** (4 группы по 4 символа, например `abcd efgh ijkl mnop`).
5. **Скопируйте его без пробелов** → `abcdefghijklmnop`. Сохраните в надёжном месте.
   - Показывается **один раз** — если потеряете, создадите новый.
6. Нажмите **«Готово» / «Done»**.

⚠️ Этот 16-значный код — секрет, как пароль. Не выкладывайте, не коммитьте в GitHub.

---

Как получите пароль приложения — напишите «готово» (сам код присылать НЕ нужно). Дальше Шаг 3.

---

## Шаг 3. Прописать данные в проект (файл `.env`)

В файле `.env` на сервере (НЕ `.env.example`, НЕ в Git) заполните SMTP-блок. Переменные уже есть в проекте:

```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш_адрес@gmail.com
SMTP_PASSWORD=пароль_приложения_16_символов_без_пробелов
SMTP_USE_TLS=true
```

Плюс адрес отправителя и получатель отчёта (в `settings.yaml`, секция email):
- `from_address` — тот же Gmail-адрес, что в SMTP_USER;
- список получателей — на кого слать отчёт (например, ваш email и/или Екатерины).

После правки — перезапустить сервис/контейнер.

---

## Шаг 4. Проверка (для разработчика)

- Сервис при еженедельной задаче (или в тестовом прогоне с `dry_run`) отправит письмо на указанный адрес.
- Признак успеха: письмо приходит, в логах нет ошибки `SMTP authentication failed`.
- Если ошибка авторизации — проверить: 2FA включена, используется именно «Пароль приложения» (не обычный), адрес в `SMTP_USER` полный.

---

## Ограничения Gmail (важно знать)

- Лимит отправки: ~500 писем в сутки (для еженедельного отчёта — с огромным запасом).
- Если в будущем понадобится массовая рассылка гостям — лучше отдельный транзакционный сервис. Для управленческих отчётов Gmail подходит полностью.
- Альтернатива: Яндекс.Почта (smtp.yandex.ru, порт 465/SSL) — тоже через пароль приложения. Принцип тот же.

---

## Что нужно от Сергея (итог)

1. Включить 2FA на Gmail (Шаг 1).
2. Создать «Пароль приложения» (Шаг 2).
3. Передать разработчику / прописать в `.env`: адрес Gmail + пароль приложения (Шаг 3).
4. Указать получателей отчёта.

После этого SMTP-часть боевого запуска (Приоритет 1) закрыта. Останется только финальная проверка Max — и можно запускать систему в бой.
