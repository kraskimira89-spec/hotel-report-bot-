## РАЗДЕЛ 1 — «Конкуренты» (`/competitors`)

Цель: единая страница со всей информацией о конкурентах — обзор, история цен, скриншоты, сравнение с нашими ценами.

### БД
Создай таблицу `competitor_prices` (миграция в `src/storage/`): `id, competitor_name, date, price_from REAL NULL, currency TEXT DEFAULT 'RUB', source TEXT (dom|vision), screenshot_path TEXT NULL, available INTEGER, created_at`.

### Хелперы в `queries.py`
- `get_competitor_latest() -> list[dict]` — последняя запись по каждому конкуренту.
- `get_competitor_history(name: str, days: int = 90) -> list[dict]`.

### Маршрут и шаблон
- `@app.get("/competitors")` → `templates/competitors.html`, `page="competitors"`.
- Отдавать скриншоты статикой только авторизованным: смонтировать `data/screenshots/` как `/static/screenshots/` (FastAPI StaticFiles) ИЛИ отдельный маршрут с проверкой auth.

### Содержание страницы
1. **Обзорная таблица** (все 9 конкурентов из config.competitors): Название · Тип (direct/indirect, цветной бейдж) · Движок (parser) · Цена «от» (последняя) · Δ к нашей цене 1apart (сопоставимая категория) · Обновлено (дата) · Статус (🟢 собрано / 🟡 из кэша / 🔴 недоступно по available) · Ссылка (target=_blank).
2. **Карточка конкурента** (раскрытие строки): история цен за 90 дней (таблица + простой график/спарклайн), скриншот виджета за последнюю дату с датой снимка, пометка способа цены (dom/vision), сопоставление категорий конкурента с нашими 6 категориями.
3. **Сводка «Мы vs рынок»**: средняя цена прямых конкурентов, наша цена, позиция (выше/ниже/в рынке, %). Отдельно прямые (Петровские, Bon Apart, Центральный, Гоголь) и косвенные (Xander, Кухтерин, Скандинавия, Парад Парк, Элегант).

### Требования
- Если `available=False` → «нет публичной цены» (не пусто).
- Сопоставление категорий — через справочник в config (competitor ↔ наши slug), не хардкод в шаблоне.
- Без персональных данных и секретов.

### Тесты
- `get_competitor_latest`/`get_competitor_history` на фикстурной БД.
- `/competitors` = 200 под авторизацией; редирект без авторизации.

## РАЗДЕЛ 2 — «Тренды» (`/trends`)

Цель: авто-радар лучших идей и трендов апарт-отелей — по России (Москва) и за рубежом. Показывает, что внедряют лидеры, какие технологии/практики повышают доход и загрузку.

### БД
Создай таблицу `trends`: `id, title, summary, category, region TEXT (ru|world), source_url, published_at DATE NULL, takeaway TEXT, is_idea_of_week INTEGER DEFAULT 0, created_at`. Ретеншен 180 дней.

### Структура данных (в `market_trends.py`)
```python
class TrendItem(BaseModel):
    title: str
    summary: str
    category: str        # из справочника ниже
    region: str          # ru | world
    source_url: str
    published_at: date | None = None
    takeaway: str        # "что это значит для нас"
    is_idea_of_week: bool = False
```

### Справочник категорий (фиксированный)
1. Технологии и ИИ · 2. Динамическое ценообразование / RMS · 3. Прямые бронирования · 4. Бесконтактный сервис · 5. Длительное проживание / корпоративные гости · 6. Гость-опыт и допуслуги · 7. Рынок и регулирование.

### Автосбор (еженедельно, отдельная задача планировщика)
Расширь `market_trends.py`: `fetch_market_trends(period, region) -> list[TrendItem]`.
- Источники — из `config.market_news.sources` (список RSS/URL отраслевых порталов) и `config.market_news.regions`.
- Способ: httpx + парсинг RSS/HTML (те же анти-блок правила, что и site_prices), затем ИИ-суммаризация в 5–8 карточек с категорией, регионом и практическим выводом.
- Сохранение в БД `trends`. Graceful-фолбэк: если источники недоступны — показать последние сохранённые + сиды (ниже).
- Добавь задачу в `scheduler.py` (еженедельно, не в ежедневном цикле).

### Маршрут и шаблон
- `@app.get("/trends")` → `templates/trends.html`, `page="trends"`.
- Блоки: **Лента карточек** (заголовок, описание, категория, регион 🇷🇺/🌍, источник-ссылка, дата, вывод «что это значит для нас»); **Фильтры** (регион ru/world/все, категория, период 7/30/90 дней); **«Идея недели»** (выделенная карточка).
- Выбор идеи: `market_news.idea_category_priority` + резервный список `idea_category_priorities` (7 категорий). `_refresh_idea_of_week()` перебирает категории по порядку, при пустоте — любой свежий тренд. Сиды в `seed_trends_if_empty()` гарантируют непустую БД.
