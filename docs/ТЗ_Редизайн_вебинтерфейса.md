# ТЗ — редизайн веб-интерфейса админки (без потери функциональности)

Документ для разработчика к проекту hotel-report-bot. Цель: навести «красоту и блеск» в админке, сохранив ВСЮ функциональность и структуру шаблонов. Работа — почти целиком в CSS.

---

## 0. Ключевой принцип

Все шаблоны наследуют `src/web/templates/base.html`, а стили — инлайн в `<style>` этого файла. Значит редизайн делается **централизованно**, через переработку CSS в `base.html`, без изменения логики маршрутов, форм и данных. HTML-классы менять только при необходимости (лучше сохранить существующие: `.card`, `.badge`, `.grid-2`, `.big`, `.muted` и др.).

Правило: **ни одна кнопка, форма, таблица, ссылка или переключатель не должны перестать работать.** Сначала внешний вид, функциональность — неприкосновенна.

---

## 1. Что не трогать (функциональность)

- Маршруты и формы (`/login`, `/settings/dry-run`, `/reports/{id}/resend` и т.д.).
- Jinja-переменные и блоки (`{% block content %}`, `{{ data... }}`).
- Названия классов, которые используются в шаблонах (переопределять стили, а не удалять классы).
- Навигацию (пункты меню и `page`-активность) — только оформление, не состав.
- Логику dry-run, авторизацию, отдачу скриншотов.

---

## 2. Фирменная дизайн-система — стиль сайта 1apart.ru

Админка должна выглядеть как часть бренда «Первый Апарт-отель». Палитра и шрифты взяты напрямую с сайта 1apart.ru (проверено по css/style.css).

Фирменный стиль сайта: премиальный «апарт-отельный» — золотисто-бежевый акцент на тёмном/белом фоне, заголовки узким гротеском Oswald.

В начало `<style>` добавить `:root` с фирменными токенами:

```css
:root {
  /* Нейтральные */
  --bg:        #FAF8F5;   /* тёплый светлый фон страницы */
  --surface:   #FFFFFF;   /* карточки */
  --surface-2: #F6F1EA;   /* вторичные поверхности (тёплый оттенок) */
  --border:    #E6DDCF;   /* границы (тёплый песочный) */
  --text:      #1B1B1B;   /* основной текст (почти чёрный, как на сайте) */
  --text-muted:#7A756C;   /* вторичный текст */
  --dark:      #121212;   /* тёмные блоки/шапка (цвет сайта) */
  --dark-brown:#3B170B;   /* фирменный тёмно-коричневый (с сайта) */
  /* Акцент — фирменное золото 1apart */
  --primary:   #C19B6A;   /* главный золотисто-бежевый (с сайта) */
  --primary-h: #A9824F;   /* hover — темнее золото */
  --primary-2: #DCB179;   /* светлое золото (градиенты) */
  --primary-soft:#F5EBDC; /* фон акцентных блоков (бледное золото) */
  /* Семантические (статусы-светофор) — согласованы с тёплой гаммой */
  --ok:   #2E7D32;  --ok-bg:  #E8F3E6;
  --warn: #B26A00;  --warn-bg:#F7EAD2;
  --err:  #B3261E;  --err-bg: #F6E2DD;
  /* Форма */
  --radius: 10px;
  --radius-sm: 6px;
  --shadow: 0 1px 2px rgba(59,23,11,.06), 0 4px 14px rgba(59,23,11,.06);
  --shadow-hover: 0 2px 6px rgba(59,23,11,.10), 0 10px 28px rgba(59,23,11,.10);
  --gap: 1rem;
}
```

Важно по акценту: золото `#C19B6A` красиво, но на белом даёт низкий контраст. Поэтому:
- золото — для плашек, границ, акцентов, активного подчёркивания, фона кнопок;
- текст-ссылки и важный текст на белом — тёмно-коричневым `--dark-brown` или `--primary-h` (для контраста WCAG AA);
- на тёмном фоне (шапка `--dark`) золото используется как акцент текста (контраст хороший).

Тёмная тема — опционально.

## 3. Типографика (как на сайте)

Сайт 1apart.ru использует **Oswald** (узкий гротеск) для заголовков и **Arial** для текста. Повторим это:
```css
@import url('https://fonts.googleapis.com/css2?family=Oswald:wght@400;500;600;700&display=swap');
body { font-family: Arial, 'Helvetica Neue', Helvetica, sans-serif; }
h1, h2, h3, .brand, .big { font-family: 'Oswald', Arial, sans-serif; letter-spacing: .01em; }
```
- Заголовки Oswald — можно с лёгким uppercase для h1/бренда (как на сайте), letter-spacing небольшой.
- Иерархия: h1 — 1.7rem/600 Oswald; h2 — 1.25rem/600 Oswald; h3 — 1rem/600; body — 15–16px/1.55 Arial; подписи — 13px muted.
- Числа в таблицах и KPI: `font-variant-numeric: tabular-nums;` (ровные колонки).
- Межстрочный интервал body 1.5, заголовки 1.2.

## 4. Компоненты — что улучшить

### 4.1 Навигация (`nav`)
- Тёмная шапка в стиле сайта: фон `--dark` (#121212), `position: sticky; top:0; z-index:10`, лёгкая тень.
- Бренд «🏨 Первый Апарт-отель» — слева, Oswald, белый/золотой, можно uppercase.
- Пункты: светло-серые (#DDD) по умолчанию; активный/hover — золотой `--primary` с подчёркивающей полосой снизу (border-bottom 2px `--primary`). Это повторяет тёмно-золотую гамму сайта.
- «Выход» — визуально отделить (справа, приглушённый).
- Мобильно: горизонтальный скролл или перенос; не ломать.

### 4.2 Карточки (`.card`)
- Фон `--surface`, `--radius`, `--shadow`, padding 1.25–1.5rem, граница 1px `--border`.
- Ховер (для кликабельных): `--shadow-hover`, лёгкий подъём `transform: translateY(-1px)` с `transition`.
- Заголовок карточки h2 — с небольшим отступом снизу.

### 4.3 KPI / крупные числа (`.big`)
- Крупно (2–2.25rem, 700), рядом подпись `--text-muted`.
- Добавить мини-«KPI-карточку»: значение сверху, лейбл снизу, при наличии — цветная дельта (↑ зелёная / ↓ красная).

### 4.4 Таблицы
- `th` — `--surface-2`, `--text-muted`, 600, uppercase 12px, letter-spacing .02em.
- Зебра строк: чётные — `--surface-2`. Ховер строки — `--primary-soft`.
- Границы только горизонтальные (`--border`), без вертикальных.
- Числа — правое выравнивание, tabular-nums.
- На узких экранах — горизонтальный скролл контейнера, не ломать вёрстку.

### 4.5 Бейджи-статусы (`.badge`)
- Скруглённые pill'ы. Семантика:
  - зелёный `--ok/--ok-bg` (собрано / выключен / успех),
  - жёлтый `--warn/--warn-bg` (из кэша / dry-run включён),
  - красный `--err/--err-bg` (недоступно / ошибка).
- Не полагаться только на цвет: добавить значок/точку (🟢🟡🔴 или ● + текст) — для доступности.

### 4.6 Кнопки и формы
- Основная кнопка: фон `--primary` (золото), текст тёмный `--dark` или белый (выбрать по контрасту), `--radius-sm`, hover `--primary-h`, `transition`. Можно лёгкий золотой градиент (`--primary`→`--primary-2`), как акцентные кнопки на сайте.
- Вторичная: контурная (border `--border`, текст `--text`).
- `.btn-small` — компактная.
- Поля ввода/select: граница `--border`, `--radius-sm`, focus — рамка `--primary` + мягкое кольцо `box-shadow: 0 0 0 3px var(--primary-soft)`.
- Формы настроек — аккуратные fieldset с заголовками-легендами.

### 4.7 Графики (bars / sparkline)
- Столбцы — `--primary` с лёгким градиентом; закруглить верх; при наведении — тултип (title) со значением.
- Sparkline на карточках конкурентов — тонкие бары `--primary`, компактно.
- Никаких 3D и лишних сеток (data-ink ratio).

### 4.8 Раздел «Тренды»
- Карточки трендов: заголовок, бейдж категории (акцентный контур), флажок региона 🇷🇺/🌍, дата muted, блок «что это значит для нас» на фоне `--primary-soft` с левой полосой `--primary`.
- «Идея недели» — выделенная карточка с рамкой-акцентом и подписью «Идея недели».

### 4.9 Раздел «Конкуренты»
- Обзорная таблица с бейджами статуса и типа (direct/indirect — разный оттенок).
- Карточка конкурента: скриншот в аккуратной рамке `--radius-sm`, история цен sparkline, «Мы vs рынок» — выделенный блок.

### 4.10 Экран входа (`login.html`)
- Центрированная карточка на фоне `--bg`, логотип/название, поля и кнопка по дизайн-системе. Аккуратная подача ошибки (`--err`).

## 5. Мелочи, дающие «блеск»
- Плавные переходы: `transition: .15s ease` на ховерах ссылок, кнопок, карточек.
- Консистентные отступы (система 4/8/12/16/24 px).
- Пустые состояния (`.muted` «Нет данных») — с иконкой/приглушённой подачей, а не голым текстом.
- `main { max-width: 1180px }`, комфортные поля.
- Favicon (🏨 или простой SVG) и аккуратный `<title>` (уже есть).
- Фокус-стили видимы (accessibility) — не убирать outline полностью, заменить на кольцо `--primary`.

---

## 6. Требования (границы работ)
- Всё делать через `<style>` в `base.html` (централизованно). Точечные правки шаблонов — только для новых обёрток KPI/статусов, без изменения данных.
- Сохранить адаптивность (уже есть `@media(max-width:768px)`), проверить на мобильном.
- WCAG AA: контраст текста ≥ 4.5:1, крупный ≥ 3:1; не полагаться только на цвет.
- Не добавлять тяжёлых JS-фреймворков. Допустимо немного ванильного JS для тултипов/раскрытия карточек (или CSS-only).
- Кросс-браузерность: свежие Chrome/Firefox/Safari.
