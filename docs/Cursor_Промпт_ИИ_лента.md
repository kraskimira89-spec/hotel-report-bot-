Ты — старший Python/фронтенд-разработчик. В существующем проекте `hotel-report-bot` (FastAPI-админка + Jinja2, SQLite, планировщик APScheduler, интеграция TravelLine) нужно добавить новый раздел **«Аналитика»** — ленту ИИ-выводов с рекомендациями. ## Главная идея
TravelLine в личном кабинете уже показывает сырые таблицы по каналам, гостям, доходу, конкурентам. **Дублировать эти таблицы НЕ нужно.** Вместо этого ИИ анализирует уже собранные данные (метрики из SQLite + данные TravelLine + цены конкурентов + тренды из интернета) и выдаёт ленту осмысленных карточек: вывод + рекомендации + кнопка «Подробнее». Это как новостная лента, но вместо новостей — аналитика.

## РАЗДЕЛ «Аналитика» (`/analytics`) — сделать стартовой страницей админки

### Бэкенд — модуль ИИ-аналитики
Создай `src/analytics/ai_insights.py` со структурой:
```python
class InsightCard(BaseModel):
    topic: str            # тема из справочника ниже
    title: str            # заголовок-вывод
    summary: str          # краткая аналитика, 1–2 предложения
    recommendations: list[str]   # 2–3 конкретные рекомендации
    severity: str         # info | attention | action
    source: str           # travelline | web | mixed
    period: str           # за какой период
    detail_payload: dict  # данные для «Подробнее» (графики/таблицы)
    updated_at: datetime
```
Темы карточек:
- Внутренние (TravelLine/метрики): загрузка и динамика; доход/ADR/RevPAR; каналы (прямые vs агрегаторы); повторные гости; отмены; средний срок проживания.
- Внешние (интернет): конкуренты (позиция по цене); тренды рынка апарт-отелей (Москва/мир); спрос в регионе (события/сезонность); регулирование.

Логика:
- Для каждой темы собери компактный контекст (цифры за период + сравнение с прошлым периодом) из уже имеющихся данных (SQLite/TravelLine/market_trends). НЕ делай новых тяжёлых запросов к TravelLine на каждый заход.
- Передай контекст в LLM с жёстким промптом: «верни JSON: {title, summary (1–2 фразы), recommendations (2–3 пункта), severity}». Парси строго в `InsightCard`.
- Батч-генерация всех тем за один прогон. Кешируй результат в БД (таблица `insights`), не дёргай LLM на каждый запрос страницы.
- Пересчёт раз в сутки (задача в `scheduler.py`, утро) + кнопка «Обновить сейчас» в разделе.
- Graceful-фолбэк: если LLM/источник недоступен — показывай последние сохранённые карточки.

### БД
Таблица `insights` (миграция в `src/storage/`): id, topic, title, summary, recommendations (JSON), severity, source, period, detail_payload (JSON), updated_at.

### Хелперы в `queries.py`
- `get_insights(source: str|None, topic: str|None) -> list[dict]` — с сортировкой по severity (action → attention → info), затем по updated_at.
- `get_top_insights(limit=2)` — «Главное сегодня».

### Маршрут и шаблон
- `@app.get("/analytics")` → `templates/analytics.html`, `page="analytics"`, `_require_auth`. Сделать её стартовой (редирект с `/` на `/analytics`).
- Кнопка/маршрут «Обновить» (`@app.post("/analytics/refresh")`) — запускает пересчёт (в фоне), затем редирект обратно.
- Пункт меню «Аналитика» в `base.html` — первым в навигации.

### Макет карточки — адаптивный ГИБРИД B+A (важно!)
- Десктоп (≥1024px): две зоны рядом — слева аналитика (заголовок + summary + «Подробнее»), справа колонка рекомендаций. CSS Grid: `grid-template-columns: 1fr 320px`.
- Узкий экран (<1024px): рекомендации переносятся ПОД блок аналитики. `@media (max-width:1024px){ grid-template-columns: 1fr; }`.
- Один HTML, поведение через `@media`. Рекомендации — на фоне `--primary-soft` с левой золотой полосой `--primary`, заголовок «💡 Рекомендации».
- Серьёзность — тонкая полоса слева карточки: 🔴 action / 🟡 attention / 🟢 info (значок + цвет, не только цвет — WCAG).
- Метка источника (TravelLine / интернет) и период — muted, мелким.

### Кнопка «Подробнее» (accordion)
- Раскрывается на всю ширину карточки (под обеими зонами): график(и) по теме из `detail_payload` (загрузка по дням, доход, доли каналов — существующие данные), таблица-детализация как подкрепление вывода, ссылки на источники (для внешних тем — URL + дата).
- Данные брать из уже собранного (SQLite), без повторных тяжёлых запросов.

### Лента и приоритизация
- Сверху блок «Главное сегодня» (1–2 карточки severity=action).
- Ниже — лента, отсортированная по severity.
- Фильтры: источник (TravelLine / интернет / все), тема, период.

### Убрать дублирование
- Разделы, которые повторяют таблицы TravelLine (детальные «Каналы» и т.п.), убрать из меню или свести к «Подробнее» внутри карточек. Оставить: Аналитика (главная), Конкуренты, Тренды, Отчёты, Логи, Настройки.

## Требования к коду
- Промпты к LLM — с жёстким JSON-форматом ответа, устойчивый парсинг в `InsightCard`.
- ИИ-выводы кешировать в БД; пересчёт по расписанию + кнопка «Обновить».
- Адаптив B+A через `@media`. Авторизация `_require_auth`. Все внешние факты — со ссылкой и датой.
- Не дублировать таблицы TravelLine — только выводы + «Подробнее» как подкрепление.

## Тесты
- Парсинг ответа LLM в `InsightCard` (замоканный JSON-ответ).
- `get_insights` сортирует по severity; фильтры работают на фикстурной БД.
- `/analytics` = 200 под авторизацией; редирект без авторизации.
- Фолбэк: при недоступном LLM показываются последние сохранённые карточки.
- Адаптив: на узком экране рекомендации в стеке (проверка разметки/классов).
