# ТЗ — раздел «Аналитика» как лента ИИ-выводов с рекомендациями

Документ для разработчика к проекту hotel-report-bot. Меняет концепцию веб-интерфейса: вместо дублирования таблиц TravelLine — лента ИИ-аналитики. Макет рекомендаций: **адаптивный гибрид B+A** (решение заказчика).

---

## 1. Идея и принцип

TravelLine в личном кабинете уже показывает сырые данные по каналам, гостям, доходу, конкурентам. **Дублировать эти таблицы в админке не нужно.** Ценность сервиса — не в повторе таблиц, а в том, что **ИИ анализирует данные (из TravelLine + из интернета) и выдаёт осмысленные выводы и рекомендации.**

Интерфейс превращается в **ленту аналитических карточек** — как новостная лента, но вместо новостей — выводы ИИ по разным темам бизнеса.

Каждая карточка = один осмысленный вывод:
- заголовок (суть вывода),
- краткая ИИ-аналитика (1–2 предложения),
- рекомендации (что делать),
- кнопка «Подробнее» → раскрывает полный отчёт по теме (графики, детализация, данные-первоисточник).

### Что меняется в структуре админки
- Новый раздел **«Аналитика»** (`/analytics`) — главная страница-лента (можно сделать её и стартовым дашбордом).
- Разделы, которые **дублируют** TravelLine (например, детальные таблицы «Каналы»), не развивать/убрать из меню — их роль берут карточки аналитики + кнопка «Подробнее». Оставить только то, что реально нужно как «сырьё» (Логи, Настройки, Отчёты).

## 2. Макет карточки — адаптивный гибрид B+A

**Широкий экран (десктоп, ≥ 1024px):** две зоны рядом.
```
┌──────────────────────────────────┐ ┌───────────────────────┐
│ 📉 Загрузка на выходные −18%       │ │ 💡 Рекомендации         │
│ [краткий вывод ИИ, 1–2 фразы]      │ │ • Снизить цену пт-сб    │
│                                    │ │   на 8–10%              │
│ [Подробнее ▼]                      │ │ • Акция «2+1»           │
└──────────────────────────────────┘ └───────────────────────┘
```

**Узкий экран (планшет/телефон, < 1024px):** рекомендации автоматически уходят ПОД блок аналитики.
```
┌──────────────────────────────────┐
│ 📉 Загрузка на выходные −18%       │
│ [краткий вывод ИИ]                 │
│ ──────────────────────────────── │
│ 💡 Рекомендации:                    │
│ • Снизить цену пт-сб на 8–10%      │
│ • Акция «2+1»                       │
│ [Подробнее ▼]                       │
└──────────────────────────────────┘
```

Реализация: CSS Grid/Flex. На десктопе — `grid-template-columns: 1fr 320px` (аналитика | рекомендации). На узком — `grid-template-columns: 1fr` (в одну колонку, рекомендации переносятся вниз). Один HTML, разное поведение через `@media`. Рекомендации — на акцентном фоне `--primary-soft` с левой золотой полосой (стиль 1apart).

Блок «Подробнее» раскрывается на всю ширину карточки (под обеими зонами): графики, таблицы-детализация, ссылки на источники.

## 3. Темы карточек (о чём аналитика)

ИИ формирует карточки по темам. Стартовый набор:

**Из данных TravelLine (внутренние):**
1. **Загрузка и её динамика** — где проседает, где пики; сравнение периодов.
2. **Доход / ADR / RevPAR** — тренд, аномалии, вклад категорий.
3. **Каналы продаж** — доля прямых vs агрегаторы, сдвиги; вывод о зависимости от каналов.
4. **Повторные гости** — доля, динамика лояльности.
5. **Отмены** — рост/падение, влияние на доход.
6. **Средний срок проживания** — сдвиги (short/long-stay).

**Из интернета (внешние):**
7. **Конкуренты** — позиция по цене (данные автосбора), кто поднял/опустил.
8. **Тренды рынка** — релевантные идеи апарт-отелей (Москва/мир), «что внедрить».
9. **Спрос в регионе** — события/сезонность, влияющие на ближайшие даты.
10. **Регулирование** — изменения законов для средств размещения/аренды.

Каждая карточка помечается: источник данных (TravelLine / интернет), период, дата обновления.

## 4. Как работает ИИ-аналитика (бэкенд)

Новый модуль, например `src/analytics/ai_insights.py`:

- **Вход:** уже собранные данные (метрики из SQLite, данные TravelLine, цены конкурентов, тренды).
- **Обработка:** для каждой темы формируется компактный контекст (цифры за период + сравнение) и передаётся в LLM с чётким промптом: «сделай краткий вывод (1–2 предложения) + 2–3 конкретные рекомендации».
- **Выход:** структура `InsightCard`:
  ```python
  class InsightCard(BaseModel):
      topic: str            # тема (из справочника п.3)
      title: str            # заголовок-вывод
      summary: str          # краткая аналитика (1–2 фразы)
      recommendations: list[str]   # 2–3 рекомендации
      severity: str         # info | attention | action (для цвета)
      source: str           # travelline | web | mixed
      period: str           # за какой период
      detail_payload: dict  # данные для «Подробнее» (графики/таблицы)
      updated_at: datetime
  ```
- **Периодичность:** пересчёт раз в сутки (утренний прогон) + возможность «обновить сейчас» из админки. Кешировать в БД (таблица `insights`), не дёргать LLM на каждый заход в интерфейс.
- **Стоимость/надёжность:** генерация батчем (все темы за один прогон), graceful-фолбэк — если LLM недоступен, показать последние сохранённые карточки.

## 5. Раздел «Подробнее» (детализация)

По клику карточка раскрывается (accordion) и показывает:
- график(и) по теме (загрузка по дням, доход, доли каналов — из `detail_payload`);
- таблицу-детализацию (уже имеющиеся данные, без дубля ЛК TravelLine — только то, что подкрепляет вывод);
- ссылки на источники (для внешних тем — URL статей/страниц конкурентов, с датой).

Данные берутся из того, что уже собрано (SQLite/TravelLine) — не делать повторных тяжёлых запросов при раскрытии.

---

## 6. Приоритизация и «лента»

- Карточки сортируются по важности: сначала `severity=action` (требует решения), потом `attention`, потом `info`.
- Вверху — «Главное сегодня»: 1–2 самые важные карточки (аналог «Идея недели»).
- Фильтры: по источнику (TravelLine / интернет / все), по теме, по периоду.
- Значок серьёзности: 🔴 действие / 🟡 внимание / 🟢 инфо (не только цвет — со значком, WCAG).

---

## 7. Стиль (фирменный 1apart)

- Использовать дизайн-систему из `docs/ТЗ_Редизайн_вебинтерфейса.md`: золото `#C19B6A`, тёмная шапка, Oswald+Arial.
- Блок рекомендаций — фон `--primary-soft`, левая полоса `--primary`, заголовок «💡 Рекомендации».
- Карточка — `--surface`, `--radius`, `--shadow`; серьёзность подсвечивается тонкой полосой слева (красная/жёлтая/зелёная).

---

## 8. Требования к коду
- Не дублировать таблицы TravelLine: показывать выводы, а сырьё — только в «Подробнее» как подкрепление.
- ИИ-выводы кешировать в БД; пересчёт по расписанию + кнопка «Обновить».
- Все внешние факты — со ссылкой на источник и датой.
- Промпты к LLM — с жёстким форматом ответа (JSON), чтобы стабильно парсить в `InsightCard`.
- Адаптив: гибрид B+A через `@media` (десктоп — 2 зоны, мобильный — стек).
- Авторизация `_require_auth`; graceful-фолбэк при недоступности LLM/источников.

## 9. Тесты
- Парсинг ответа LLM в `InsightCard` (на замоканном ответе).
- Рендер `/analytics` = 200 под авторизацией; сортировка по severity; фильтры.
- Адаптив: проверка, что на узком экране рекомендации в стеке (можно снапшот-тест разметки).
- Фолбэк: при пустом/недоступном LLM показываются последние сохранённые карточки.

## 10. Порядок реализации
1. Модуль `ai_insights.py` + структура `InsightCard` + таблица `insights` в БД.
2. Промпты по темам п.3, батч-генерация, кеш.
3. Раздел `/analytics` + шаблон `analytics.html` (гибрид B+A) + пункт меню.
4. «Подробнее» (accordion с графиками из detail_payload).
5. Задача планировщика (ежедневный пересчёт) + кнопка «Обновить».
6. Убрать/свернуть дублирующие таблицы из меню. Тесты. PR «Раздел Аналитика (ИИ-лента)».
