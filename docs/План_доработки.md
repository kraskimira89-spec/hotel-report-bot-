## ПРИОРИТЕТ 1 — Довести до боевого запуска (этап 11)

Цель: система реально шлёт корректные сводки Екатерине. Это важнее любых украшений.

1. **Подключить реальные доступы** (действия заказчика/Сергея):
   - Google: сервисный аккаунт + JSON-ключ, дать таблице доступ «Читатель».
   - TravelLine: ключ API (тариф WebPMS СТАНДАРТ+).
   - Max: chat_id уже есть (364502022); проверить webhook на проде.
   - SMTP для email.
2. **Прогон в dry_run=true, 1–2 недели**: сводки идут в тестовый чат/на свою почту.
3. **Сверка цифр** сервиса с отчётом TravelLine «Доходность и загрузка» (Occupancy/ADR/RevPAR/каналы).
4. **Донастройка порогов светофора** (% загрузки) в config по факту.
5. Проверить сценарии: расхождение источников, недоступность источника, повторная отправка из админки.
6. **Переключить dry_run=false** после подтверждения. Отметить этап 11 в README.

Результат: рабочая ежедневная Max-сводка + еженедельный email.
