# Пошаговый план настройки TravelLine API

Инструкция для Сергея (по шагам, с указанием, куда нажимать). Проверено по документации TravelLine (июль 2026). Для не-программиста: делаем медленно, после каждого блока — подтверждение.

---

## 0. Что важно понять заранее

У TravelLine ДВА разных доступа, они настраиваются по-разному и нужны для разных данных. В нашем сервисе используются оба:

| Доступ | Что даёт | Авторизация | Где берётся |
|---|---|---|---|
| **A. Universal API WebPMS** | Доход, платежи, аналитика по броням, ADR/RevPAR (эталон цифр) | Заголовок `X-API-KEY` (один ключ) | «Управление отелем» → «Настройки» → «Интеграции» |
| **B. Read Reservation API** | Список броней, источник/канал бронирования, гости | OAuth2 (`client_id` + `client_secret`) | «Настройки гостиницы» → «Подключения API» |

Минимальные требования: действующая подписка **TL: WebPMS тариф «СТАНДАРТ» и выше**. Если тариф ниже — Universal API не заработает (сначала пункт 1).

---

## Шаг 1. Проверить тариф подписки

1. Войдите в личный кабинет TravelLine (тот, где ведётся шахматка апарт-отеля).
2. Убедитесь, что подписка **TL: WebPMS** уровня «СТАНДАРТ» или выше.
3. Если не уверены — напишите в поддержку TravelLine: **support@travelline.ru** (спросите: «Какой у нас тариф TL: WebPMS и доступен ли Универсальный API?»).

⛳ Блокер: без тарифа СТАНДАРТ+ доступ A (Universal API) не выдаётся. Доступ B (Reservation API) обычно доступен и на других тарифах, но раздел «Подключения API» иногда нужно попросить включить в поддержке.

---

## Шаг 2. Доступ A — ключ Universal API WebPMS (X-API-KEY)

1. В личном кабинете перейдите: **«Управление отелем» → «Настройки»**.
2. Откройте вкладку **«Интеграции»**.
3. Найдите блок **«Доступ к Универсальному API TL: WebPMS»**.
4. Скопируйте **ключ интеграции** (длинная строка). Это и есть `X-API-KEY`.
5. Сохраните его в надёжном месте (это секрет, как пароль).

Проверка (по желанию, для разработчика): запрос
`GET https://partner.tlintegration.com/api/webpms/v1/analytics/services`
с заголовком `X-API-KEY: <ключ>` должен вернуть данные (код 200).

Если блока «Доступ к Универсальному API» нет — напишите в support@travelline.ru с просьбой включить Универсальный API (укажите тариф СТАНДАРТ+).

---

## Шаг 3. Доступ B — подключение Reservation API (client_id / client_secret)

1. В личном кабинете перейдите: **«Настройки гостиницы» → «Подключения API»**.
   - Если раздела нет — напишите в **support@travelline.ru**: «Просьба открыть доступ к разделу Подключения API».
2. Нажмите **«Создать подключение»**.
3. В поле **«Название подключения»** введите:
   ```
   Отчётность 1apart
   ```
4. На вкладке **«Публичные API»** поставьте галочки:
   - ✅ **Read Reservation API** (обязательно — список броней, каналы).
   - ✅ **Content API** (желательно — категории номеров/тарифы для сопоставления).
5. Вкладку **«Вебхуки»** пока не трогаем (не обязательно для отчётности).
6. Нажмите **«Сохранить»** → в окне «Сохранение подключения API» нажмите **«Сохранить и открыть доступ»**.
7. После сохранения автоматически появятся два поля:
   - **client_id** (логин),
   - **client_secret** (пароль).
8. Скопируйте **оба значения** и сохраните в надёжном месте (это секреты).

Примечание: на одно средство размещения можно создать максимум 3 подключения.

---

## Шаг 4. Передать ключи в проект (безопасно)

Все три значения прописываются в файле `.env` на сервере (НЕ в Git). Переменные уже предусмотрены в проекте (`config/.env.example`):

```
# Доступ A — Universal API WebPMS
TL_API_KEY=<ключ X-API-KEY из шага 2>

# Доступ B — Reservation API (OAuth2)
TL_CLIENT_ID=<client_id из шага 3>
TL_CLIENT_SECRET=<client_secret из шага 3>
```

- Базовый адрес API: `https://partner.tlintegration.com` (уже в конфиге).
- После правки `.env` — перезапустить сервис/контейнер.

⚠️ Точные имена переменных сверить с `config/.env.example` в репозитории (если отличаются — привести к тому, что в проекте).

## Шаг 5. Технические детали для разработчика (справочно)

**Авторизация Universal API (доступ A):**
- Заголовок каждого запроса: `X-API-KEY: <ключ>`.
- Content-Type: `application/json`.
- Методы: `/api/webpms/v1/analytics/services` (начисления/доход), `/analytics/payments` (платежи), `/analytics/services/cancelled` (отмены).
- Параметры: `startDate`/`endDate` (yyyyMMdd), `dateKind` (для «новых броней сегодня» = 2). Макс. окно 31 день.

**Авторизация Reservation API (доступ B, OAuth2):**
- Получение токена: `POST https://partner.tlintegration.com/auth/token`, тело `x-www-form-urlencoded`:
  `grant_type=client_credentials&client_id=...&client_secret=...`
- В ответе `access_token` (JWT). Передавать в каждом запросе: `Authorization: Bearer <access_token>`.
- Токен живёт **15 минут**, refresh не поддерживается — по истечении запрашивать новый. **Кешировать токен** и переиспользовать.
- Лимиты авторизации: 3 запроса/сек, 15/мин, 300/час с одного IP.

Оба доступа уже заложены в `src/data_sources/travelline.py`.

---

## Что нужно от Сергея (итог)

Собрать и передать разработчику (или прописать в `.env` самому):
1. **TL_API_KEY** — из «Управление отелем → Настройки → Интеграции» (шаг 2).
2. **TL_CLIENT_ID** и **TL_CLIENT_SECRET** — из «Настройки гостиницы → Подключения API» (шаг 3).
3. Подтверждение тарифа СТАНДАРТ+ (шаг 1).

После этого TravelLine-часть боевого запуска (Приоритет 1) закрыта. Останутся SMTP (email) и финальная проверка Max.
